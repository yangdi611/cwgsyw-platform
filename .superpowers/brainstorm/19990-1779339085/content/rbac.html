动态权限系统设计

超级管理员可配置角色权限，新增模块自动纳入权限管理

RBAC 数据模型
资源 (Resource)
模块名称 + 操作列表
例：change_doc → [create, read, update, approve, export]
新增模块时，在此注册资源和操作即可
↕
权限 (Permission)
resource + action 的组合
例：change_doc:approve
↕
角色 (Role)
关联 N 个 Permission
超级管理员在页面上动态配置
角色名称也可自定义，不限于预设5种
↕
用户 (User)
关联 N 个 Role，权限取并集
超级管理员配置界面
角色：运维组员 编辑权限
☑ 日报：填写、查看本人
☑ 变更文档：创建、查看
☐ 变更文档：审批
☑ 设备密码：按分类查看
☐ 设备密码：管理
新增模块权限自动出现在此列表
新增模块时的流程
1. 后端注册新模块资源 + 操作列表
2. 前端权限配置页自动渲染新权限项
3. 默认所有角色无新模块权限（最小权限原则）
4. 超级管理员按需分配
5. CLAUDE.md 提醒审查权限配置
CLAUDE.md 强制检查点
每次新增或修改模块时，必须：
① 在 Resource 表注册新资源和操作
② 检查前端权限配置页是否正确渲染
③ 检查后端 @PreAuthorize 注解覆盖所有接口
④ 更新默认角色权限说明文档
