组成员管理放在哪里？

当前 /groups 页面只有组名称+描述的表格。要加上成员管理功能，交互入口放哪里？

A

组详情弹窗 ✅ 推荐

在组表格操作列加"成员"按钮 → 点击弹出 Dialog → 左侧当前成员列表，右侧可搜索添加/移除。不跳转，一个弹窗搞定所有操作。

优点

  • 不跳转，操作流畅
  • 与项目已有的 Dialog 模式一致
  • 可以同时看到成员和待添加用户

缺点

  • 成员多时弹窗可能显小
B

独立组成员页面

点击组名或"成员"按钮 → 跳转 /groups/[id]/members → 独立页面管理成员。页面可以更大，适合成员多的场景。

优点

  • 空间大，适合大量成员
  • URL 可分享

缺点

  • 页面跳转打断操作流
  • 需要新建路由和页面
  • MVP 阶段过度设计
C

表格行内展开

点击展开箭头 → 表格行下方展开成员子列表 → 行内操作添加/移除。不弹窗、不跳转。

优点

  • 极快访问
  • 不遮挡上下文

缺点

  • 实现复杂度高
  • 成员多时表格布局乱
  • 项目无现成模式
