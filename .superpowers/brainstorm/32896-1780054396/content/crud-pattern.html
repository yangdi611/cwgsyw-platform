用户/组管理的增删改 UI 用什么交互模式？

目前 /users 和 /groups 都是纯只读表格。需要加上新建、编辑、删除功能。

A

Dialog 弹窗模式 ✅ 推荐

新建/编辑用 Modal Dialog 弹出表单，删除用 Confirm Dialog。保持在同一页面，不跳转。表格行右侧加操作按钮（编辑/删除）。

优点

  • 不离开上下文
  • 快速操作，体验流畅
  • 与项目现用 shadcn/ui Dialog 一致

缺点

  • 表单复杂时 dialog 显得拥挤
B

独立页面模式

新建/编辑跳转到独立页面（如 /users/new, /users/[id]/edit），表单可以占满整个页面。删除用 Confirm Dialog。

优点

  • 表单空间充足，适合复杂字段
  • URL 可分享

缺点

  • 页面跳转打断操作流
  • 代码量大
  • 对于简单 CRUD 过度设计
C

行内编辑模式

点击编辑按钮后表格行变为可编辑状态，直接在行内修改字段。新建用顶部空白行或小 Dialog。

优点

  • 极致快速，无需弹窗

缺点

  • 表格行内编辑复杂字段体验差
  • 实现复杂度高
  • 项目中没有现成模式可复用
