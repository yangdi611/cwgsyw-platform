# 拓扑对比模式使用指南

> 如何用 CMDB 拓扑对比模式查看 CI 实例拓扑在两个时间点的差异
> 关联 AC-6 功能：`?compare=1` 自动激活 + 实例详情入口

---

## 快速开始

### 入口

有两种方式进入拓扑对比模式：

**方式一：实例详情页拓扑 Tab**

在实例详情页的"拓扑图" Tab 中，点击右上角的 **对比模式** 链接（带 `GitCompare` 图标）：

```
┌─────────────────────────────────────────────┐
│  🎯 拓扑图                   对比模式  全屏展开 → │
└─────────────────────────────────────────────┘
```

**方式二：直接访问 URL**

在拓扑图页面的 URL 后追加 `?compare=1` 参数（页面会自动激活对比模式）：

```
/cmdb/topology/{instanceId}?compare=1
```

### 基本操作

1. 进入对比模式后，设置 **起始时间 (From)** 和 **结束时间 (To)**
2. 点击 **加载对比** 按钮发起对比查询
3. 系统会加载该时间窗口内的拓扑变更，以颜色标注差异：

| 颜色 | 含义 | 说明 |
|------|------|------|
| 🟢 绿色 | 新增 (added) | 该时间窗口内新增的节点/边 |
| 🔴 红色 | 删除 (removed) | 该时间窗口内被删除的节点/边 |
| 🟡 琥珀色 | 修改 (modified) | 属性或状态发生变化的节点 |
| ⚪ 灰色 | 未变 (unchanged) | 没有变化的节点/边 |

4. 页面左下角的图例显示当前差异状态说明：

```
🟢 新增  🔴 删除  🟡 修改  ⚪ 未变
```

## 使用场景

### 场景 1：变更审计

部署前和部署后各截取一个时间点的拓扑快照进行对比，验证变更是否符合预期。

> **操作示例**：
> 1. 部署前记下当前时间 `T1`
> 2. 完成变更操作
> 3. 打开对比模式，设置 From=T1, To=当前时间
> 4. 查看新增/修改/删除的节点，确认变更范围

### 场景 2：故障排查

怀疑某次变更导致了故障，对比故障前后的拓扑变化。

> **操作示例**：
> 1. 获取故障发生前的时间点 `T1`
> 2. 设置 From=T1, To=故障时间
> 3. 观察 removed 和 modified 的节点，定位可能的问题源

### 场景 3：运维巡检

定期对关键 CI 实例做拓扑对比，监控拓扑结构的变化趋势。

## 参数说明

| 参数 | 类型 | 默认值 | 说明 |
|------|------|--------|------|
| `from` | ISO 日期时间 | 空 | 对比起始时间，格式 `YYYY-MM-DDTHH:mm` |
| `to` | ISO 日期时间 | 空 | 对比结束时间 |
| `depth` | 数字 | 3 | BFS 遍历深度，控制对比范围 |

**注意**：不设置时间范围时，点击"加载对比"不会发送请求。必须同时填入 from 和 to。

---

## 注意事项

1. **时间精度**：对比基于 audit_log 回放，日志记录精度影响对比准确性
2. **深度限制**：对比深度默认 3 层，过深的拓扑对比可能造成较大的后端压力
3. **性能**：包含大量节点的拓扑对比可能需要数秒加载时间
4. **无数据提示**：如果时间窗口内没有变更记录，对比结果将显示空白的差异列表
5. **切换模式**：关闭对比模式会恢复到当前时刻的实时拓扑视图

## FAQ

**Q: 为什么点击"对比模式"后没有自动加载？**
A: `?compare=1` 只自动激活对比模式 UI，不自动发起对比查询 — 需要手动设置时间范围后点击"加载对比"按钮。

**Q: 对比模式下的节点是否可交互？**
A: 可以。对比模式下节点同样支持点击展开/折叠、拖拽和缩放。

**Q: 对比结果是否可导出？**
A: 对比结果可以通过拓扑页面的 PNG 导出功能一起导出。
