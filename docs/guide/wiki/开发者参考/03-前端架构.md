# 前端架构

## 技术栈

- Next.js 16 App Router + React 19 + TypeScript
- TanStack Query v5（数据获取/缓存）
- `@uiw/react-md-editor`（编辑器）
- `react-markdown` + `remark-gfm` + `remark-breaks` + `rehype-highlight`（渲染）
- `@xyflow/react`（知识图谱，与 CMDB 拓扑共用）

## 路由与页面

| 路由 | 文件 | 职责 |
|------|------|------|
| `/wiki` | `wiki/page.tsx` | 空间卡片列表 + 新建空间对话框 |
| `/wiki/[spaceId]` | `[spaceId]/layout.tsx` + `page.tsx` | 三栏布局 + 空间首页（最近更新） |
| `/wiki/[spaceId]/[pageId]` | `[pageId]/page.tsx` | 阅读页 + 右侧面板 |
| `/wiki/[spaceId]/[pageId]/edit` | `edit/page.tsx` | Markdown 编辑器 |
| `/wiki/[spaceId]/graph` | `graph/page.tsx` | 知识图谱 |
| `/wiki/search` | `search/page.tsx` | 全文搜索（Suspense 包裹 useSearchParams） |

## 组件

| 组件 | 职责 |
|------|------|
| `WikiTreeSidebar` | 递归页面树，节点操作（新建子页/重命名/删除/移动），状态圆点 |
| `WikiBacklinksPanel` | 反向链接列表 |
| `WikiVersionsPanel` | 版本历史 + 回滚 + 导出（可折叠） |
| `WikiAclDialog` | ACL 编辑（继承开关 + role/group/user + read/write/delete/publish） |

## API 层

`src/lib/wiki-api.ts` 导出 `wikiApi`，封装全部端点。

- `src/lib/api.ts` **无响应拦截解包**，故方法用 `.then(r => r.data.data)` 取 `R<T>.data`
- 导出用 `window.open('/api/wiki/...')`
- 附件上传用 `FormData`

## 类型

`src/types/wiki.ts` 全部接口字段为 **snake_case**（匹配后端 JSON）：`WikiSpace`、`WikiPageTree`、`WikiPage`、`WikiBacklink`、`WikiVersion`、`WikiSearchResult`、`WikiGraph`、`WikiAcl`、`WikiAclEntry`，外加 `PageResult<T>`。

## 权限守卫

- 页面级 `useEffect` 检查 `hasPermission('wiki', 'read')`，无权 redirect
- 按钮级守卫：
  - 编辑/树操作 → `hasPermission('wiki', 'update')`（**注意：全局动词是 `update` 不是 `write`**）
  - 删除 → `wiki:delete`
  - 发布 → `wiki:publish`
  - ACL 面板 → `wiki:manage_acl`

> 实现期曾误用 `wiki:write`（不存在的全局权限），已修正为 `wiki:update`。`write` 仅是页面 ACL 内部动词，由后端判定。

## 导航

`components/layout/Sidebar.tsx` 已加顶级「知识库」项（`BookOpen` 图标，`wiki:read`，href `/wiki`）。

## 双向链接渲染

阅读页加载页面树构建 `Map<title, {id, space_id}>`，正则把 `[[title]]` / `[[title|alias]]` 替换为指向 `/wiki/{spaceId}/{pageId}` 的链接；未匹配标题渲染为「待创建」提示。

## 编辑器要点

- `MDEditor` 通过 `dynamic(() => import(...), { ssr: false })` 动态加载（避免 SSR 报错）
- 30s debounce 自动保存 + `Ctrl+S` 手动保存
- 粘贴/拖拽图片 → `wikiApi.uploadAttachment` → 插入 `![](url)`
- `[[` 触发搜索浮层补全
