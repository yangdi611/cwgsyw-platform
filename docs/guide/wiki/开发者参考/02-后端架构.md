# 后端架构

## 服务职责

| 服务 | 职责 |
|------|------|
| `WikiSpaceService` | 空间 CRUD，删除前校验空间为空 |
| `WikiPageService` | 核心：页面树、CRUD、保存（版本+链接重建）、级联删、移动防环、版本回滚、搜索、发布/提交审批、审批回调、图谱组装 |
| `WikiBacklinkService` | `[[ ]]` 正则解析、反向链接重建与查询 |
| `WikiAclService` | 页面 ACL 覆盖式继承判定、读写 ACL（移植自 SharedFolderAclService） |
| `WikiAttachmentService` | 附件上传到 MinIO、流式代理、按页查附件 |
| `WikiExportService` | 单页 .md/.zip、批量空间 .zip（ZipOutputStream 流式） |
| `WikiPublishListener` | Flowable 结束事件回调，更新状态 + 通知 |

## 核心调用链

### 保存页面（写路径）

```
PUT /pages/{id}
 └ WikiController.savePage  (checkAcl write)
    └ WikiPageService.savePage  @Transactional
       ├ requirePage / 校验非 archived
       ├ pageMapper.updateById   (content, title, version+1)
       ├ saveVersion             → wiki_page_version 快照
       ├ backlinkService.rebuild → 解析[[]], 重建 wiki_backlink
       └ auditLogMapper.insert   (action=update)
```

> 链接解析在 content 仍在内存时完成，不回读数据库。

### 删除页面（级联）

```
DELETE /pages/{id}
 └ WikiPageService.deletePage  @Transactional
    ├ pageMapper.findDescendantIds(id)   递归 CTE 取自身+子孙
    └ for each: pageMapper.deleteById(id) + audit   ← @TableLogic 软删
```

### 提交审批 → 回调

```
POST /pages/{id}/submit
 └ runtimeService.startProcessInstanceByKey("wiki_publish", "wikiPage:{id}", {pageId,submitterId})
    → status=review
 ... 审批人在 /workflow/tasks 完成任务(approved,comment) ...
 └ 流程结束事件 → wikiPublishListener.notify
    └ WikiPageService.handleApprovalResult → status=published/draft + 通知
```

## 关键 Mapper SQL

`WikiPageMapper` 含 4 个自定义 `@Select`（均不涉及 JSONB，安全）：

- `findAncestorChain` — 递归 CTE，自身→根（ACL 继承用）
- `findDescendantIds` — 递归 CTE，自身→子孙（级联删用）
- `search` — ILIKE 匹配 + substring 摘要（仅 published）
- `searchCount` — 搜索总数（分页用）

其余表用 `BaseMapper` + `LambdaQueryWrapper`。

## 权限两层校验

1. `@PreAuthorize("hasAuthority('wiki:xxx')")` — 全局 RBAC
2. `checkAcl(user, pageId, perm)` → `WikiAclService.hasPermission` — 页面级 ACL（admin 绕过）
