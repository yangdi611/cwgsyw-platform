# 扩展指引

常见二次开发场景的落地路径。

## 1. 中文全文分词搜索

当前搜索用 ILIKE。要升级为真正的分词检索：

1. 安装 PostgreSQL `pg_jieba`（或 `zhparser`）扩展
2. 新增迁移：建分词配置 + 在 `wiki_page` 上建 GIN 索引
   ```sql
   CREATE INDEX idx_wiki_page_fts ON wiki_page
     USING GIN(to_tsvector('jiebacfg', coalesce(title,'')||' '||coalesce(content,'')));
   ```
3. 改 `WikiPageMapper.search` 用 `@@ to_tsquery('jiebacfg', ...)` + `ts_headline`
4. 保留 ILIKE 作为兜底

## 2. 页面关联 CMDB CI 实例

仿 `ChangeDocCiLink` 模式（变更文档↔CI）：

1. 新表 `wiki_page_ci_link(page_id, ci_instance_id, ...)`
2. 新服务方法 + 端点 `POST/DELETE /pages/{id}/ci-links`
3. 阅读页加「关联资产」面板

## 3. 实时协同编辑

当前为单人编辑 + 自动保存。协同需引入 WebSocket/CRDT（如 Yjs），超出现有架构，需评估。可先做「编辑锁」轻量方案：保存时校验 `current_version`，冲突则提示。

## 4. 页面模板

新建页面时套用预设模板：

1. 新表 `wiki_template(name, content)` 或复用 sys_config
2. `CreatePageRequest` 加 `template_id`
3. `createPage` 用模板内容初始化 content

## 5. 导出为 PDF/Word

当前导出 Markdown。要导出 PDF/Word，复用变更文档的 `ExportService`（pandoc）：

1. 先把 Markdown 渲染/转换
2. 复用 `MinioStorageService` 或直接流式返回

## 6. 大批量异步导出

当前批量导出同步。页面/图片很多时改异步：

1. 新表 `wiki_export_task(status, file_path, ...)`
2. `@Async` 后台打包，完成后写记录 + 站内信通知
3. 参考 `BackupService` 的任务模式

## 通用注意（务必遵守平台约定）

- 新增写操作必须同事务写 `audit_log`（`module=wiki`）
- 新增端点必须加 `@PreAuthorize` + 必要时 ACL 校验
- 软删表删除用 `deleteById`，勿手动 set isDeleted
- JSONB 字段查询用 `LambdaQueryWrapper`，勿用 `@Select`
- 新增 RBAC 动作要补迁移 seed（sys_resource.actions + sys_permission + sys_role_permission）
- 前端请求体/响应用 snake_case
