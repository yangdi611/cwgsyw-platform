# 发布审批工作流

Wiki 的发布审批复用平台现有 Flowable 工作流引擎。**后端代码已就绪**（`WikiPublishListener` 已部署），但流程定义需管理员在线配置一次。

## 设计决策

按实施计划的关键决策，BPMN 流程定义**不入代码仓库**，而是通过在线 BPMN 编辑器配置。好处：改流程零部署（无需 rebuild backend），与现有「日报审批」管理方式统一。

## 配置步骤

1. 进入 **工作流 → 流程管理**（`/workflow/admin`）
2. 新建流程，**流程 key 必须为 `wiki_publish`**（与后端 `submitForReview` 中 `startProcessInstanceByKey("wiki_publish", ...)` 一致，不可改名）
3. 设计流程：
   - 开始事件
   - 用户任务（`userTask`）：设置候选组 `flowable:candidateGroups`（如审批组）
   - 结束事件：挂执行监听器
4. **结束事件必须挂监听器**引用已部署的 Bean：

```xml
<endEvent id="end">
  <extensionElements>
    <flowable:executionListener event="start" delegateExpression="${wikiPublishListener}"/>
  </extensionElements>
</endEvent>
```

5. 部署并激活流程

## 流程变量约定

| 变量 | 类型 | 写入时机 |
|------|------|---------|
| `pageId` | Long | 提交审批时由后端写入 |
| `submitterId` | Long | 提交审批时由后端写入 |
| `approved` | Boolean | 审批人完成任务时写入（通过/拒绝） |
| `comment` | String | 审批人完成任务时写入（可选意见） |

## 运行时流转

```
用户「提交审批」
  → 后端 startProcessInstanceByKey("wiki_publish", "wikiPage:{id}", {pageId, submitterId})
  → 页面 status=review, 记录 process_instance_id
  → 审批人在 /workflow/tasks 处理（复用现有审批页，零改动）
  → 完成任务写入 approved/comment
  → 结束事件触发 wikiPublishListener
      ├─ approved=true  → status=published + 通知创建者
      └─ approved=false → status=draft + 通知创建者（含拒绝意见）
```

## 注意

- 若流程未配置/未激活，「提交审批」会因找不到流程定义而失败；此时可改用直接发布（给相应角色分配 `wiki:publish`）
- `WikiPublishListener` 已随后端部署，无需额外操作
- 审批完成后的状态变更由系统自动执行（审计日志 `operatorId=0` 表示系统操作）
