# 附件与搜索端点

## POST /api/wiki/attachments

上传附件（图片）。权限 `wiki:update` + 页面 ACL `write`。`multipart/form-data`。

**表单字段**
| 字段 | 说明 |
|------|------|
| `file` | 文件（MultipartFile） |
| `page_id` | 所属页面 ID |

**响应**
```json
{ "code": 200, "data": { "file_id": 15, "url": "/api/wiki/attachments/15" } }
```

直接把 `url` 以 `![](url)` 插入 Markdown 即可。

```bash
curl -X POST http://localhost:8081/api/wiki/attachments \
  -H "Authorization: Bearer $T" \
  -F "file=@/path/img.png" -F "page_id=1"
```

后端将文件上传到 MinIO（key=`wiki/<spaceId>/attachments/<uuid>.<ext>`），并在 `shared_file` 表登记（`source_type=wiki_page`）。

---

## GET /api/wiki/attachments/{fileId}

流式下载/展示附件。权限 `wiki:read`。

- 后端从 MinIO 读取并流式写出（非 302 预签名）
- 响应头：`Content-Type` 按扩展名推断（png/jpg/gif/webp/svg），`Cache-Control: private, max-age=86400`
- 可直接用于 `<img src="/api/wiki/attachments/15">`

---

## GET /api/wiki/search

全文搜索（ILIKE）。权限 `wiki:read`。

**查询参数**
| 参数 | 默认 | 说明 |
|------|------|------|
| `keyword` | — | 关键词（标题或正文，不区分大小写） |
| `space_id` | 可选 | 限定空间 |
| `page` | 1 | 页码 |
| `size` | 20 | 每页条数 |

**响应**
```json
{
  "code": 200,
  "data": {
    "records": [
      { "page_id": 1, "space_id": 1, "title": "部署手册",
        "highlight": "...关键词周边摘要...", "updated_at": "..." }
    ],
    "total": 1, "page": 1, "size": 20
  }
}
```

- **仅检索 `published` 页面**
- `highlight` 为关键词周边 `substring` 摘要片段
- 结果不含完整正文（避免大字段传输）

```bash
# 中文关键词需 URL 编码
curl "http://localhost:8081/api/wiki/search?keyword=%E9%83%A8%E7%BD%B2&page=1&size=10" \
  -H "Authorization: Bearer $T"
```
