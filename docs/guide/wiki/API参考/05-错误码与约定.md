# 错误码与约定

## 响应包装

所有 JSON 端点返回统一包装 `R<T>`：

```json
{ "code": 200, "message": "success", "data": <T> }
```

流式端点（导出、附件下载）直接返回文件流，不走 `R<T>` 包装。

## 状态码

| code | 含义 | 触发场景 |
|------|------|---------|
| 200 | 成功 | 正常 |
| 401 | 未认证 | 缺失/过期 JWT |
| 403 | 无权限 | 缺少 `wiki:*` 全局权限，或页面 ACL 不通过（`AccessDeniedException`） |
| 409 | 冲突 | 删除非空空间（`空间非空，请先删除全部页面`） |
| 500 | 服务器错误 | 非法状态（如归档页保存、回滚不存在的版本、移动到子孙节点） |

## 认证

```
Authorization: Bearer <JWT>
```

获取 token：

```bash
curl -X POST http://localhost:8081/api/auth/login \
  -H "Content-Type: application/json" \
  -d '{"username":"superadmin","password":"Admin@123"}'
# → data.token
```

## 字段命名

- 请求体与响应体统一 **SNAKE_CASE**（后端全局 Jackson 配置）
- 即 Java 的 `spaceId` ↔ JSON 的 `space_id`

## 业务约束速查

| 约束 | 端点 |
|------|------|
| 删空间前须清空页面 | DELETE /spaces/{id} |
| 删页面级联删子孙 | DELETE /pages/{id} |
| 仅 draft 可提交审批 | POST /pages/{id}/submit |
| 归档页不可保存 | PUT /pages/{id} |
| 不可移动到自身子孙 | POST /pages/{id}/move |
| 搜索/图谱仅含 published | GET /search, /spaces/{id}/graph |
| slug 同级唯一（冲突追加后缀） | POST /pages |

## 双向链接语法

正文中：
- `[[标题]]` — 链接并显示「标题」
- `[[标题|别名]]` — 链接到「标题」，显示「别名」

保存时按标题精确匹配现有页面建立 `wiki_backlink`；未匹配的标题不建链接（前端渲染为「待创建」提示）。
