# 搜索

## 全文搜索

入口：顶部搜索框或 `/wiki/search?keyword=...`。

- 在标题和正文中匹配关键词（不区分大小写）
- 返回结果列表：页面标题、关键词周边摘要片段、最近更新时间
- 点击结果跳转到对应页面
- 支持分页

## 搜索范围

- **仅检索 `published` 状态的页面** —— 草稿和审批中的页面不会出现在搜索结果
- 跨空间检索（你有 `wiki:read` 权限的范围内）
- 可选按空间过滤（`space_id` 参数）

## 实现说明（为何用 ILIKE）

搜索基于 PostgreSQL `ILIKE '%关键词%'` 子串匹配：

- 对中文友好（PostgreSQL 默认 `tsvector` 的 `simple` 配置对中文无分词能力）
- 与平台 CMDB 搜索保持一致的技术路线
- 摘要片段由 `substring(content, 关键词位置-50, 100)` 截取生成

> 未来若页面规模显著增长，可引入 `pg_jieba` 中文分词扩展并改用 GIN 全文索引。当前内部团队体量下 ILIKE 全表扫描性能足够。
