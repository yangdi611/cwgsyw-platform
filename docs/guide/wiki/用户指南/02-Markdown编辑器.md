# Markdown 编辑器

Wiki 使用 `@uiw/react-md-editor` 作为编辑器，纯 Markdown 工作流（与 Obsidian 一致），不是 WYSIWYG。

## 进入编辑

阅读页右上角 **编辑** 按钮（需 `wiki:update` 权限 + 页面 ACL `write`），或在页面树新建页面后自动进入。

## 界面

- **标题输入框**：位于编辑器上方，独立于正文
- **编辑区 / 预览区**：默认分栏（live preview），左侧 Markdown 源码，右侧实时渲染
- **状态栏**：显示保存状态（如「已保存 16:42:05」）

## 支持的 Markdown 语法

基于 `remark-gfm`（GitHub Flavored Markdown）+ `rehype-highlight`：

- 标题 `#`~`######`
- 粗体 `**x**`、斜体 `*x*`、删除线 `~~x~~`
- 有序/无序列表、任务列表 `- [ ]` / `- [x]`
- 表格
- 代码块（带语法高亮）、行内代码
- 引用 `>`
- 链接、图片
- 双向链接 `[[页面标题]]`（见 [双向链接](./03-双向链接与知识图谱.md)）

## 图片上传

两种方式，均上传至 MinIO 对象存储：

1. **粘贴**：截图后直接 `Ctrl+V`，自动上传并在光标处插入 `![](/api/wiki/attachments/{id})`
2. **拖拽**：将图片文件拖入编辑区

> 图片通过后端流式代理访问（`GET /api/wiki/attachments/{id}`），无需 MinIO 对外暴露。

## 自动保存

- 停止输入 **30 秒** 后自动静默保存，状态栏更新时间戳
- 每次保存（含自动保存）都会生成一个版本快照

## 快捷键

| 快捷键 | 作用 |
|--------|------|
| `Ctrl+S` | 手动保存（可附版本备注） |
| `[[` | 触发页面链接自动补全浮层 |

## 保存时发生了什么

保存（`PUT /api/wiki/pages/{id}`）在同一数据库事务内完成：

1. 更新正文与标题，版本号 +1
2. 解析正文中所有 `[[链接]]`，重建反向链接关系
3. 写入版本快照
4. 写审计日志（`module=wiki`, `action=update`）
