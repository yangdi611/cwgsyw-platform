# 空间与页面管理

## 空间（Space）

空间是知识库的顶级容器。一个团队可以按业务域划分多个空间（如「运维手册」「网络架构」「应急预案」）。

### 创建空间

1. 左侧导航栏点击 **知识库**
2. 右上角 **新建空间**（需要 `wiki:create` 权限）
3. 填写：
   - **名称**（必填）
   - **描述**（可选）
4. 确定后空间卡片出现在列表，点击 **进入**

### 空间列表

每张空间卡片显示：名称、描述、页面数量、最近更新时间、创建人。

### 删除空间

- 入口：空间卡片或空间内操作菜单
- **约束：空间必须为空**（无任何页面）才能删除，否则返回提示「空间非空，请先删除全部页面」
- 删除为软删除，数据保留在数据库中（`is_deleted=true`）

---

## 页面（Page）

页面是承载 Markdown 内容的文档，按树形结构组织，支持无限层级嵌套。

### 页面树

进入空间后，左侧为可折叠的页面树（宽约 260px）。每个节点显示：

- 文档/目录图标
- 页面标题
- 状态圆点：草稿（灰）/ 审批中（琥珀）/ 已发布（绿）

### 新建页面

- 在页面树某节点的操作菜单选择 **新建子页**，或在空间根新建顶级页面
- 仅需填写 **标题**；系统自动生成 URL 友好的 `slug`（同级唯一，冲突时追加后缀）
- 新页面初始为 **草稿（draft）**，创建后自动跳转到编辑器

### 重命名页面

操作菜单 → 重命名。系统会先拉取当前内容再更新标题，避免误清空正文。

### 移动 / 排序页面

- **移动**：修改页面的父节点（操作菜单或拖动）
- **排序**：同级内通过 `sort_order` 调整顺序（上移/下移）
- **防环保护**：不能把页面移动到它自己的子孙节点下，系统会拒绝并提示

### 删除页面

- 软删除（`@TableLogic` 逻辑删除）
- **级联**：删除一个页面会同时软删除它的所有子孙页面
- 删除前如有子页面，界面会提示子页数量供确认
- 删除后从页面树消失，但反向链接记录保留（其他页面若引用了被删页，阅读时显示「(已删除)」）

---

## 状态机

页面在生命周期中流转于以下状态：

| 状态 | 含义 | 可执行操作 |
|------|------|-----------|
| `draft` | 草稿 | 编辑、提交审批、直接发布 |
| `review` | 审批中 | 等待审批人处理（不可编辑提交） |
| `published` | 已发布 | 可被搜索、出现在知识图谱；可再次编辑 |
| `archived` | 已归档 | 只读，不可编辑 |

> 全文搜索与知识图谱**仅包含 `published` 状态**的页面。
