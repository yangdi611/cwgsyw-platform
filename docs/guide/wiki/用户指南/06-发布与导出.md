# 发布与导出

## 发布

页面默认是 **草稿（draft）**，需要发布才能被搜索和出现在知识图谱中。发布方式取决于你的权限：

### 方式一：直接发布（有 `wiki:publish` 权限）

- 页面右上角显示 **发布** 按钮
- 点击后状态直接变为 `published`
- 发布成功会给页面创建者发一条站内信通知

### 方式二：提交审批（无 `wiki:publish` 权限）

- 页面右上角显示 **提交审批** 按钮
- 仅 `draft` 状态可提交
- 提交后：
  1. 状态变为 `review`（审批中）
  2. 启动 Flowable 工作流（流程 key = `wiki_publish`）
  3. 审批人在 **工作流 → 我的任务**（`/workflow/tasks`）处理
  4. 审批通过 → 状态变 `published` + 通知创建者
  5. 审批拒绝 → 状态回到 `draft` + 通知（含拒绝意见）

> 工作流流程需管理员在 BPMN 编辑器中预先配置，详见 [管理员指南](../管理员指南/03-发布审批工作流.md)。

## 导出

所有导出产物均为 Markdown 文档，图片一并打包。

### 单页导出

入口：阅读页 / 版本面板的 **导出** 按钮（`GET /api/wiki/pages/{id}/export`）。

- **页面无图片** → 直接下载 `.md` 文件
- **页面含图片** → 下载 `.zip`：
  ```
  页面标题.md          ← 图片引用改写为 ./images/xxx
  images/
    <图片文件>
  ```

### 批量导出（整个空间）

入口：空间内 **导出** 按钮（`GET /api/wiki/spaces/{id}/export`）。

- 下载 `.zip`，**保留页面树的目录层级**：
  ```
  部署手册.md
  网络拓扑.md
  部署手册/
    子章节.md
  images/
    <所有页面去重后的图片>
  ```
- 文件名中的非法字符（`\ / : * ? " < > |`）会被替换为 `_`
- 图片全局去重（同一图片只打包一份）
- 中文文件名以 UTF-8 编码写入 zip（用支持 UTF-8 的解压工具打开）

### 导出技术说明

- 纯 Java `ZipOutputStream` 流式写入响应，不在服务器落盘
- 图片从 MinIO 实时读取打包
- 大批量（>50 页）当前为同步导出；若导出很慢，管理员可参考部署文档为导出端点放宽 nginx 超时
