# cwgsyw-platform 测试用例

## 使用说明

本文档包含平台全部模块的手工测试用例，按模块组织。每条用例包括：ID、优先级、前置条件、测试步骤、预期结果、清理、备注。

### 优先级说明
- **P0** — 核心流程，阻塞上线，必须全部通过
- **P1** — 重要功能/边界/权限，本迭代修复
- **P2** — 体验优化，可评估延期
- **P3** — 建议项，排入后续迭代

### 执行要求
- P0 用例：严格按照每一步操作执行并核对预期结果，不得跳跃
- P1 用例：至少按场景描述验证主流程
- P2 用例：选择性执行

### 用例模板

每条用例采用如下格式：

| 字段 | 值 |
|------|-----|
| **ID** | TC-AUTH-001 |
| **优先级** | P0 |
| **前置条件** | ... |
| **测试步骤** | 1. ... 2. ... |
| **预期结果** | 1. ... 2. ... |
| **清理** | ... |
| **备注** | ... |

### TC-AUTH-001：正常登录

| 字段 | 值 |
|------|-----|
| **ID** | TC-AUTH-001 |
| **优先级** | P0 |
| **前置条件** | 1. 后端服务已启动，数据库已初始化（V3 种子数据包含 superadmin）<br>2. 前端服务已启动，可访问 `http://localhost:3000`<br>3. 浏览器清除 localStorage/cookies |
| **测试步骤** | 1. 打开浏览器，访问 `http://localhost:3000`<br>2. 确认页面跳转到 `/login`<br>3. 在用户名输入框输入 `superadmin`<br>4. 在密码输入框输入 `Admin@123`<br>5. 点击"登录"按钮 |

| **清理** | 在右上角用户菜单中点击"退出登录"，确认跳转回 `/login` |
| **备注** | 首次登录或 seed 后执行，确保数据正确 |
