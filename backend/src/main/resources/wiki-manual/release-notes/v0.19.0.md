# v0.19.0 — 2026年6月28日

本次更新围绕**硬件、机房与数据库**的配置管理展开，新增多类内置模型、机柜 2D 视图、维保状态自动提醒与批量编辑能力。

## 新增功能

### 硬件 / 数据库 / 机房内置模型

「模型管理」中新增多类开箱即用的内置模型，覆盖日常运维的主要资产类型：

- **硬件** — 存储、SAN 交换机、网络交换机、机柜
- **数据库与中间件** — MySQL、OceanBase、Oracle、Redis、Kafka、Nginx
- **机房** — 楼宇、机房

每类模型都预置了常用属性，涵盖基础信息、硬件规格、维保信息等，并已配置好资产之间的包含与运行关系，在拓扑图中自动呈现上下游依赖，无需从零搭建。

### 机柜 2D 视图

机柜实例详情页新增「机柜视图」：

- 按 U 位真实定位每台设备，直观看到机柜里装了什么、还剩多少空间
- 设备块可点击，直达对应设备详情
- 提供「添加设备」入口，快速把设备装入机柜
- 自动检测 U 位越界、区间重叠、未登记 U 位等问题并在顶部提示
- 已关联但未填 U 位的设备集中列出，避免遗漏

### 维保状态自动提醒

所有模型新增「维保信息」，其中维保状态会根据到期日期自动计算并在详情页以醒目徽章呈现：

- 在保（绿）
- 即将到期（黄）
- 已过保（红）

未填写到期日期的设备不显示徽章。

### 批量编辑

实例列表页新增多选与批量编辑：勾选多条实例后，可一次性把某个字段更新为统一的新值。每条记录独立留痕，单条失败不影响其余，结束后汇总成功与失败明细。

## 优化

- 补全资产之间的关联关系，机柜包含设备、应用与数据库运行在主机上等依赖关系在拓扑图中可自动呈现
- 云资源池模型补齐与其他模型一致的维保信息及容量配置属性
