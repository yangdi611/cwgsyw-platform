# v0.27.0 — 2026年7月2日

本次更新聚焦「账号安全」和「用户管理」：新用户首次登录必须完成安全设置，用户可以自助维护资料和修改密码，管理员也可以更可靠地重置密码、禁用账号和强制下线。此前一些登录态不同步、旧密码复用、账号禁用后仍可继续访问的问题，本版一并修复。

## 新增

- **首次登录安全设置** — 新用户首次登录后会进入「完善账号安全」页面，必须修改初始密码并补全邮箱、手机号后才能继续使用系统。这样可以避免初始密码长期使用，也能让通知、排班、找回和审计等账号信息更完整。
- **个人资料自助维护** — 顶部头像菜单新增「个人资料」入口，用户可自行维护邮箱、手机号和头像地址；用户名和真实姓名仍由管理员统一维护，避免关键身份信息被误改。
- **自助修改密码** — 顶部头像菜单新增「修改密码」入口，用户输入当前密码后即可自行更换密码，日常改密不再必须联系管理员。
- **管理员重置密码** — 用户管理页新增「重置密码」操作。管理员重置后，用户当前登录会立即失效，下次登录必须重新修改密码，适合账号交接、密码泄露排查和离岗处理。
- **管理员强制下线** — 管理端支持撤销指定用户的当前登录状态；禁用用户、删除用户、重置密码时也会同步让该用户重新登录，避免账号状态已变更但旧页面仍可继续操作。
- **密码安全规则** — 创建用户、管理员重置密码、首次登录改密、自助改密都会检查密码强度，并提示具体不满足的规则。新密码不能包含用户名，也不能复用初始密码或近期使用过的密码。
- **空闲自动退出** — 用户长时间没有真实操作时，系统会自动退出登录，减少共享电脑、临时离开工位后账号被继续使用的风险。

## 优化

- **账号菜单更完整** — 原先顶部只展示头像和退出按钮，现在头像菜单集中提供个人资料、修改密码和退出登录，账号相关操作更容易找到。
- **用户创建流程更清晰** — 新建用户时支持填写手机号；邮箱、手机号可由管理员预填，也可留给用户首次登录时补全。创建成功后会明确提示「首次登录需修改密码并补全资料」。
- **用户编辑更聚焦** — 编辑用户时不再把「改资料」和「改密码」混在同一个表单里，密码统一通过「重置密码」处理，降低误改密码的风险。
- **用户列表支持搜索** — 用户管理列表支持按用户名、真实姓名、邮箱搜索，人员多时查找账号更快。
- **会话状态更一致** — 多个页面同时打开时，退出登录、会话失效、密码重置等状态会同步到当前浏览器窗口，减少一个页面已退出、另一个页面仍显示可用的情况。
- **登录安全信息更完整** — 用户账号会记录最近登录时间、最近登录 IP、密码修改时间和资料完整状态，方便后续安全排查和账号治理。

## 修复

- **账号禁用后旧登录仍可继续使用** — 此前管理员禁用账号后，用户已打开的页面可能仍保留旧登录状态。本版修复后，禁用、删除、重置密码等管理动作都会让对应账号的旧登录失效。
- **旧登录状态无法被主动撤销** — 过去登录状态主要依赖浏览器本地保存，管理员很难让某个用户立即重新登录。本版补齐了服务端会话管理，异常、超时、被撤销的登录状态都会被明确拦截。
- **首次登录流程容易被绕过** — 需要改初始密码或补全资料的用户，现在只能访问账号设置相关页面；完成前访问其他业务页面会被引导回设置页。
- **密码重复使用风险** — 修复了新密码可以和初始密码或近期旧密码重复的问题，降低简单轮换后继续使用旧密码的风险。
- **资料不完整不易发现** — 邮箱、手机号缺失的账号现在能被明确识别，并可按配置要求引导用户补全。
- **运维日历管理菜单显示异常** — 修复主日历页管理下拉菜单的分组显示问题，排班、节假日、模板、统计、素材等入口展示更稳定。
