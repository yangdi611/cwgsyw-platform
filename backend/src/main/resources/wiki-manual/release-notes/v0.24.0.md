# v0.24.0 — 2026年6月29日

本次更新修复了设备密码库的登录身份解析缺陷，并将设备库筛选维度升级为 CMDB 模型分组，同时补全了凭据操作的组级越权防护。

## 修复

- **设备密码全部不可见** — 2026-06-28 全局 camelCase 改造遗漏了 `useAuth.ts`，导致登录后 `groupScope`/`groupId` 解析失败，所有用户（包括 superadmin）被误判为"组级身份"，设备详情页凭据区块渲染空白。现已修复解构字段名与后端保持一致（camelCase），**修复后请退出重新登录**以刷新浏览器缓存的身份状态。

## 优化

- **设备库按 CMDB 模型分组筛选** — `/devices` 页搜索栏旁的类型筛选从写死的 5 类（服务器/网络/安全/云资源/其他）改为从 CMDB 模型分组（`ci_model_group`）动态获取。筛选项只显示当前库中真实有设备的分组（如硬件设备、数据库、中间件…），后续在 CMDB 新增模型并录入设备后，筛选项自动跟随，无需额外配置。设备列表新增「模型分组」列，方便一眼识别归属。

## 安全

- **凭据跨组越权写操作防护** — `addCredential`（创建账号）和 `deleteCredential`（删除账号）此前缺少组级越权校验，组级用户（组长/组员）可绕过前端限制操作其他组的凭据。现已在后端加入与 `reveal`（查看密码）相同的组级校验：组级用户只能操作本组凭据，admin/superadmin 不受限制。
