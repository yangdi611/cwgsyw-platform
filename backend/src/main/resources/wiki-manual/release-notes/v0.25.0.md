# v0.25.0 — 2026年6月30日

本次更新上线全新的「运维日历」模块，把日报、巡检、排班、季报、半年报、密码更新、监控分析等周期性运维事项统一纳入「计划、提醒、执行、留痕、复盘」的闭环；并将工作台主卡片从「近期 CMDB 变更」替换为「运维日历」。

## 新增

- **运维日历（一级菜单）** — 侧边栏新增「运维日历」一级入口，提供月历、周视图、列表视图三种视图。点击日期可打开当日工作项的放大视图，点击任务打开详情面板。普通成员默认看「我的」、组长默认看「本组」、管理员默认看「全部」，各看各的、互不干扰。
- **任务闭环** — 每条任务都有清晰的流转状态：待确认 → 未开始 → 进行中 → 已完成，另有已逾期、异常关闭、已取消。负责人可以确认收到、开始执行、提交完成，提交时可填写执行结论、风险等级、勾选检查项，并关联相关的日报、配置项、告警、变更文档。必填检查项没完成不允许提交，确保该走的流程不被跳过。
- **周期规则** — 管理员可以创建按每日、每周、每月、每季度、每半年等节奏自动生成任务的规则，省去人工每次手动建任务。设置时可直接预览未来几次的触发时间，做到心中有数。平台还内置了日报提醒、下周巡检、季报、季度核查、季度密码更新、半年报归集等常用规则，开箱即用。
- **排班与节假日历** — 可维护值班排班，支持设置备份负责人、覆盖联系方式，并在排班冲突或缺少手机号时给出提醒；可维护节假日历，包含调休补班日，作为工作日推算的依据。
- **通知与升级** — 任务生成、临近到期、迟迟未确认、已经逾期，都会通过站内信加邮件提醒到人；长时间未确认或逾期的任务会自动升级提醒到上级。同一件事不会反复打扰，避免通知轰炸。
- **报表素材归集** — 提供季度、半年的素材归集入口，自动汇总任务完成情况、类型分布与关联对象，为撰写季报、半年报准备好现成素材。

## 优化

- **工作台卡片替换** — 工作台主区域的「近期 CMDB 变更」卡片替换为「运维日历」卡片，直接展示今日待办、待确认、逾期数量与最近任务，更贴合「今天该做什么」的日常使用习惯。CMDB 变更记录仍可在 CMDB 模块内查看。
- **日报提醒迁移** — 原先单独的日报提醒整合进运维日历统一管理。日报仍作为重要的周期任务保留，并作为周报、月报、季报、半年报的基础数据；采用每天一条汇总跟踪、再向未提交人单独提醒的方式，既不漏人，也不会让日历被大量重复条目淹没。
