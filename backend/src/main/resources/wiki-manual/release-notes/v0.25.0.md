# v0.25.0 — 2026年6月30日

本次更新上线全新的「运维日历」模块，把日报、巡检、排班、季报、半年报、密码更新、监控分析等周期性运维事项统一纳入「计划、提醒、执行、留痕、复盘」的闭环；并将工作台主卡片从「近期 CMDB 变更」替换为「运维日历」。

## 新增

- **运维日历（一级菜单）** — 侧边栏新增「运维日历」一级入口，提供月历、周视图、列表视图三种视图，点击日期打开当日工作项放大视图，点击任务打开详情抽屉。普通成员默认看「我的」、组长默认看「本组」、管理员默认看「全部」。
- **任务闭环** — 任务支持完整状态机：待确认 → 未开始 → 进行中 → 已完成，以及已逾期、异常关闭、已取消。负责人可确认收到、开始执行、提交完成（含执行结论、风险等级、SOP 检查项、关联日报/CI/告警/变更文档），必填检查项未完成不可提交。
- **周期规则** — 管理员可创建按每日/每周/每月/每季度/每半年/cron 触发的周期规则，自动生成日历任务。表单内可结构化编辑触发配置并预览未来 6 次触发时间。内置「日报未提交提醒、下周巡检、季报、季度核查、季度密码更新、半年报归集」等规则模板（仅日报提醒默认启用）。
- **排班与节假日历** — 维护值班排班（含备份负责人、联系方式覆盖、冲突检测与缺少手机号提醒）和节假日历（含调休补班日、工作日推算基础）。
- **通知与升级** — 任务生成、到期、未确认、逾期均通过站内信+邮件提醒，逾期/未确认自动升级给升级人；通知按「任务+阶段+用户+渠道」幂等，不重复发送。
- **报表素材归集** — 提供季度/半年素材清单归集入口，汇总任务完成情况、类型分布与关联对象，为季报/半年报提供素材。

## 优化

- **工作台卡片替换** — 工作台主工作区的「近期 CMDB 变更」卡片替换为「运维日历」卡片，展示今日待办、待确认、逾期数与最近任务，更贴合「今天该做什么」的高频使用场景。CMDB 变更仍可在 CMDB 模块内查看。
- **日报提醒迁移** — 旧的硬编码日报提醒调度器停用，迁移为运维日历内置规则。日报数据作为周/月/季/半年报的基础数据保留为一等任务类型，采用「每天一条汇总跟踪任务 + 向未提交者扇出通知」策略，既不降级又避免日历被淹没。

## 技术

- 新增 Flyway `V61__ops_calendar.sql`：11 张表（规则/任务/参与人/检查项/日志/关联/通知幂等/模板/排班/节假日）+ `ops_calendar` RBAC 资源与权限种子。
- 所有写操作在同一事务内写平台 `audit_log`（合规审计）与领域 `ops_schedule_task_log`（业务时间线），并存不互替；调度器系统操作 `operatorId=0`。
- 调度器每分钟 tick，生成阶段按 `next_generate_at <= now` 命中部分索引过滤，不做全表扫描；通知模板渲染全程 null-safe，缺失字段降级为占位或省略，不输出 `null`。
