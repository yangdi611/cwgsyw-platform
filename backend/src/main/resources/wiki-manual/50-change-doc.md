# 变更文档

变更文档用于规范化管理 IT 变更：从填写变更申请、走审批流程，到通过后自动归档，全程留痕。

## 什么时候用

任何对生产环境的重要改动（上线、配置修改、扩容、下线等）都应走变更文档，确保：
- 变更有计划、有评审
- 有人审批把关
- 留下完整记录可追溯

## 新建变更文档

进入 **变更文档 → 新建变更**：

1. 选择变更模板（不同类型变更有不同模板字段）
2. 按动态表单填写变更内容（变更原因、方案、回滚计划、影响范围等）
3. 可关联相关 [[CI 实例管理|CMDB 配置项]]，标明这次变更涉及哪些资产
4. 保存草稿或提交审批

## 审批流程

提交后进入审批工作流：

- 审批人收到通知
- 审批人查看变更详情，决定通过或驳回
- 结果通知提交人
- 通过后状态变为"已发布"

## 自动归档

变更文档审批通过后：

1. 系统自动生成 Word 和 PDF 文档
2. 归档到[[共享文件库]]（按年月分目录，如「变更文档 / 2026 / 06」）
3. 归档文件的可见范围继承变更的审批组

这样每次变更都有正式文档存档，无需手动整理。

## 模板管理

管理员可在"模板管理"中维护变更模板：

- 定义模板字段（类型、必填、选项等）
- 不同变更场景用不同模板

需要 `change_doc_template` 相关权限。

## AI 辅助

部分字段支持 AI 辅助填写，帮助快速起草变更方案（取决于系统是否配置了 AI 服务）。

## 权限

| 权限 | 能做什么 |
|------|---------|
| `change_doc:create` | 创建变更 |
| `change_doc:read` | 查看 |
| `change_doc:update` | 编辑 |
| `change_doc:approve` | 审批 |

相关：[[CMDB 配置管理]]、[[共享文件库]]
