# 关联与拓扑视图

关联描述配置项之间的关系，拓扑视图把这些关系可视化，帮助你理解基础设施的依赖结构。

## 建立关联

在某个 CI 实例的详情页 → 关联面板：

1. 点"新建关联"
2. 选择关联类型（语义）和目标 CI
3. 保存

关联是**有方向**的（从源 CI 指向目标 CI），系统会校验该类型关联是否允许在这两个模型间建立，以及基数约束（1:1 / 1:n / n:n）。

## 关联类型（语义）

平台内置以下关联种类：

| 类型 | 含义 |
|------|------|
| 属于 | 包含/从属关系 |
| 运行 | 运行/被运行 |
| 连接 | 网络或逻辑连接 |
| 依赖 | A 依赖 B |
| 部署 | A 部署在 B 上 |

## 拓扑视图

进入某实例的拓扑全屏页：

- 以图形方式展示该 CI 及其关联的其他 CI
- 可选择**遍历深度**（1-5 层），看多远的关联
- 点击节点查看详情
- 支持按类型过滤、导出 PNG 图片

### 对比模式
可选择两个时间点，对比拓扑结构的变化（新增/删除/修改的节点和关联），用于排查"什么时候多了/少了一个依赖"。

## 影响分析

当某个 CI 要变更或下线时，影响分析帮你回答"动它会影响哪些其他系统"：

- 从目标 CI 出发，沿关联向外遍历
- 列出所有受影响的 CI

这是变更前评估风险的重要工具。需要 `cmdb_instance:impact` 权限。

## 2D 分组视图

对启用了 2D 视图的模型，可按某个属性分组，以网格方式展示实例，适合按机房、环境等维度俯瞰资源分布。
