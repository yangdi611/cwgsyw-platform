# 告警中心

告警中心接收与配置项相关的告警（监控系统推送或手动录入），并进行分类处理与跟踪。

## 告警来源

- **外部系统推送**：通过 API 将监控系统（Zabbix、Prometheus 等）的告警推送到平台
- **手动创建**：运维人员手动录入发现的问题

## 告警列表

进入 **CMDB → 告警中心**：

- 列表展示所有告警，按严重度（critical / high / medium / low）标注颜色
- 多维过滤：按状态、严重度、时间、关联 CI、操作人
- 支持关键词搜索

## 告警字段

| 字段 | 说明 |
|------|------|
| 关联 CI | 与哪个配置项相关（必填） |
| 严重度 | critical / high / medium / low |
| 状态 | open（待处理）/ in_progress（处理中）/ resolved（已解决）/ closed（已关闭） |
| 标题 | 简短描述 |
| 详情 | 详细内容、告警消息原文 |
| 创建者 | 谁录入的 |
| 处理人 | 当前负责处理的人 |

## 告警处理

1. 点击某条告警进详情
2. 可以：
   - 更新状态（认领 → 处理中 → 已解决）
   - 添加备注（记录处理过程）
   - 关联其他 CI（如问题扩散）
   - 关闭告警
3. 所有操作都有记录

## 告警统计

告警中心首页展示：
- **今日新增 / 待处理 / 已解决** 数量
- **趋势图表** — 每日新增/解决告警数
- **Top 告警 CI** — 哪些配置项最频繁报警

## 通知

重要告警可配置站内信或邮件通知（需管理员配置）。

## 权限

- `cmdb_alert:create` — 创建告警
- `cmdb_alert:update` — 处理告警
- `cmdb_alert:delete` — 删除告警（一般仅管理员）
