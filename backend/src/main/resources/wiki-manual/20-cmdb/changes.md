# 变更历史与统计

CMDB 自动记录每个配置项的修改，方便追溯"谁在什么时候改了什么"。

## 变更记录

进入 **CMDB → 变更记录**：

- 全局变更时间线，按时间倒序展示所有 CI 的改动
- 多维过滤：按实体类型、模型、操作人、时间范围、操作类型（新建/修改/删除）
- 分页浏览

在单个实例的详情页，也能查看该实例自己的变更历史。

## 能看到什么

每条变更记录包含：

- 操作人
- 操作时间
- 操作类型（创建 / 更新 / 删除）
- 改动的具体字段（变更前 → 变更后）

## 变更统计

进入 **CMDB → 变更统计** 面板：

- **今日 / 本周 / 本月** 变更数量卡片
- **每日趋势** 图表
- **Top 10 活跃实例**（改动最频繁的 CI）

统计帮助你了解配置数据的变动节奏，识别异常频繁的改动。

## 用途

- **审计追溯** — 出问题时回查是谁、什么时候改的
- **变更复盘** — 配合[[变更文档]]核对实际改动
- **活跃度监控** — 通过统计发现异常
