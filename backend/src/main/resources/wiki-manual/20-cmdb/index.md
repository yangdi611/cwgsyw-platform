# CMDB 配置管理

## 什么是 CMDB

CMDB (Configuration Management Database) 用于统一管理 IT 基础设施配置项（CI），例如服务器、应用、网络设备等，并记录它们之间的关联关系。它是整个平台的数据基础——设备、变更、日报等模块都可以关联到 CMDB 中的配置项。

## 核心概念

### 配置项（CI）
一个可管理的 IT 资源实体，例如一台服务器、一个应用系统。每个 CI 有：
- **唯一标识**（实例 ID）
- **所属模型**（例如"主机"、"应用"）
- **属性**（IP 地址、负责人、操作系统等）
- **关联**（与其他 CI 的依赖或从属关系）

### 模型（Model）
定义一类 CI 的结构。例如"主机"模型规定了主机类 CI 应该有哪些字段（内网 IP、主机名、操作系统、环境等）。平台内置「主机」「应用」两个模型，管理员可新增。

### 关联（Relation）
描述两个 CI 之间的关系，例如"应用 A 部署在主机 B 上"、"应用 C 依赖应用 D"。关联是有方向的，并按语义分类（属于、运行、连接、依赖、部署等）。
