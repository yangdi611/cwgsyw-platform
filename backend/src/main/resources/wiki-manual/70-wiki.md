# Wiki 知识库使用说明

Wiki 知识库用于团队知识沉淀，支持多空间、树形页面、Markdown 编辑、双向链接、知识图谱和版本历史。你正在阅读的这份手册就在 Wiki 里。

## 空间与页面

- **空间（Space）** — 顶级隔离单元，类似一个独立的知识库（如"运维手册"、"项目文档"）
- **页面（Page）** — 空间内的文档，可以有父子层级，形成目录树

进入 **Wiki** 看到所有空间卡片，点击"进入"打开某个空间。

## 创建内容

> 注意：本「平台使用手册」空间是系统维护的只读空间，普通用户不能编辑。下面说明针对你自己创建的空间。

- **新建空间**：在 Wiki 首页点"新建空间"
- **新建页面**：在空间内的左侧树点 + 号，可建顶级页或子页
- **编辑页面**：进入页面点"编辑"，用 Markdown 编写
- **移动页面**：在树中拖拽调整层级和顺序

## Markdown 编辑器

页面正文用 Markdown 编写，支持标题、列表、表格、代码块、图片等。编辑时自动保存草稿。

## 双向链接

用 `[[页面标题]]` 语法引用其他页面：

- 输入 `[[` 会触发自动补全
- 链接会渲染成可点击的跳转
- 被链接的页面能看到"哪些页面引用了我"（反向链接面板）

例如本手册中的 [[CMDB 配置管理]]、[[设备密码库]] 都是双向链接。

## 知识图谱

每个空间可查看知识图谱：以图形展示页面之间的链接关系，直观看到知识结构和孤立页面。

## 版本历史

每次保存都会留下版本快照：

- 在页面查看"版本历史"
- 可对比、回滚到任意历史版本

## 发布与审批

页面有状态：草稿 → 审批中 → 已发布。

- 普通空间的页面可提交审批，审批通过后发布
- 发布后才会出现在知识图谱中

## 搜索

Wiki 提供全文搜索，跨空间或限定空间查找页面。

## 导出

支持把单个页面或整个空间导出，便于离线保存或分享。

## 权限

| 权限 | 能做什么 |
|------|---------|
| `wiki:read` | 浏览 |
| `wiki:create` | 创建空间/页面 |
| `wiki:update` | 编辑 |
| `wiki:delete` | 删除 |
| `wiki:publish` | 发布 |
| `wiki:manage_acl` | 配置页面访问权限 |

还可对单个页面设置 ACL（继承式），用法类似[[共享文件库]]的文件夹权限。
